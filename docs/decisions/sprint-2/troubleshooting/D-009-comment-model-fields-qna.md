## Final Accepted Prompt

통과된 Decision:
- D-009. 댓글 모델은 어떤 필드를 가질 것인가?

사용자 최종 답변:

```md
선택: C

이유: Sprint 2에서 댓글 수정 기능도 포함하므로 수정 시각 추적이 필요하다.

댓글 권한 판단 기준: `comments.user_id == current_user.id`

표시용 작성자 정보 저장 여부: 아직 프로필 정책이 없으므로 댓글 row에는 저장하지 않는다.

DB/API/테스트 영향: `comments` 테이블이 추가되고 `updated_at`을 포함한다. 댓글 API와 기본 API 테스트가 추가된다.

아직 다음 분기로 남겨둘 것: 댓글 API 계약, 조회 방식, 수정/삭제 endpoint와 권한 적용 방식.

롤백 또는 재검토 조건: 사용자가 선택이 완전히 잘못됐다고 판단하거나 재학습/재진입이 필요하다고 판단하면 재검토한다.
```

Codex 평가:
- Pass

Pass 이유:
- C 선택이 명확하다.
- 댓글 수정 기능 포함이라는 사용자 의도와 `updated_at` 포함 모델이 일치한다.
- 댓글 권한 기준은 `user_id`로 정리됐다.
- `post_id`는 본인 판단 값이 아니라 게시글 스코프 확인값으로 분리됐다.
- 작성자 표시 정보는 현재 중복 저장하지 않는다.

보완 질문 여부:
- 있음.
- A 선택과 댓글 수정 포함 의도가 충돌해 C로 변경할지 확인했다.

아직 남은 후속 분기:
- D-010. 댓글 API 계약은 어떻게 둘 것인가?
- 댓글 수정/삭제 권한 적용 계층과 테스트 범위
- 최소 화면 구현 범위

최종 결론:
- 댓글 모델은 `id`, `post_id`, `user_id`, `content`, `created_at`, `updated_at`을 가진다.
- 구현은 D-010 API 계약이 Pass된 뒤 시작한다.
