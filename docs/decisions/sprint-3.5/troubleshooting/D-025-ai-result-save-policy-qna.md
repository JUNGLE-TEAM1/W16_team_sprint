## Final Accepted Prompt

통과된 Decision:
- D-025. AI 결과 저장 정책을 어떻게 둘 것인가?

사용자 최종 답변:

```md
선택: A

이유:
UX 측면에서 A가 맞다.

게시글 저장 흐름 반영 방식:
Agent 결과는 바로 DB에 저장하지 않고 미리보기로만 반환한다. 사용자가 반영 버튼을 누르면 작성 폼의 `title`, `content`, `tags`에 채워지고, 최종 저장은 기존 게시글 작성 API를 통해 사용자가 직접 수행한다.

API/UI/테스트 영향:
API는 Agent 미리보기 응답을 반환하고 게시글 저장은 기존 작성 API를 유지한다. UI는 AI 미리보기와 반영 버튼이 필요하다. 테스트는 Agent 결과가 자동 저장되지 않고, 사용자 반영 후 기존 작성 흐름으로 저장되는지 확인한다.

아직 다음 분기로 남겨둘 것:
프론트엔드 미리보기 UI 상세, 반영 버튼 문구, Agent 결과 DB 저장 여부의 후속 확장, 외부 참고자료 표시 방식, AI 생성 표시 문구, 사용 이력 저장 여부는 후속 구현 또는 Sprint 4 세부 작업으로 남긴다.
```

Codex 평가:
- 선택지가 명확하다.
- 게시글 저장 흐름 반영 방식이 설명되었다.
- API/UI/테스트 영향이 언급되었다.
- 후속 분기가 구분되었다.

Pass 이유:
- Level 3 Decision의 사용자 선택, 이유, 영향 범위, 후속 분기 구분이 충족되었다.

보완 질문 여부:
- 있음.

아직 남은 후속 분기:
- Sprint 3.5의 구현 전 필수 Level 3 분기는 모두 통과했다.
- Sprint 4 구현 중 세부 UI 문구, model 구체값, threshold, JSON-RPC envelope 등은 하위 분기로 다룬다.

최종 결론:
- AI 결과는 미리보기로만 반환한다.
- 사용자가 확인하고 반영해야 작성 화면에 채워진다.
- 최종 저장은 기존 게시글 작성 API를 통해 사용자가 직접 수행한다.
