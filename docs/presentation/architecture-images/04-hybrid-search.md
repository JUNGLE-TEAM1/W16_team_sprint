# 04. Hybrid Search 구조 발표 대본

## 슬라이드 한 줄 요약

벡터 검색과 키워드 검색을 함께 사용해 조선왕조실록 기사 후보를 찾습니다.

## 발표 대본

이 슬라이드는 실록 기사 검색 구조입니다.

사용자가 역사 질문을 입력하면 먼저 Query rewrite/filter 단계를 거칩니다. 여기서는 질문에서 왕 이름, 연도, 사건, 주제 같은 검색 단서를 정리하고, 검색에 불필요한 표현은 줄입니다.

그 다음 검색은 두 갈래로 나뉩니다.

위쪽은 Vector Search입니다. 사용자의 질문을 embedding으로 바꾸고, `annals_chunks`에 저장된 vector와 비교해서 의미적으로 가까운 chunk를 찾습니다.

아래쪽은 Keyword Search입니다. 실록 검색에서는 왕 이름, 날짜, 사건명처럼 정확한 단어가 중요한 경우가 많기 때문에 제목, 본문, subject class를 대상으로 키워드 검색도 함께 수행합니다.

두 검색 결과는 Score Merge 단계에서 합쳐집니다. 같은 `article_id`에 대해 벡터 점수와 키워드 점수를 조합해서 최종 기사 후보를 만듭니다.

이렇게 만든 Article 후보는 다음 단계인 MCP 원문 조회와 LLM rerank로 넘어갑니다.

이 구조의 장점은 의미 기반 검색과 명시적 키워드 검색을 같이 쓴다는 점입니다. 그래서 질문 표현이 조금 달라도 관련 기사를 찾을 수 있고, 동시에 왕 이름이나 사건명처럼 정확히 맞아야 하는 검색도 놓치지 않게 됩니다.

## 짧게 말할 때

Hybrid Search는 embedding으로 의미가 가까운 기사를 찾고, keyword로 왕 이름이나 사건명 같은 명시적 단서를 보강한 뒤, 두 결과를 합쳐 최종 기사 후보를 만드는 구조입니다.
