# 01. 전체 시스템 구조 발표 대본

## 슬라이드 한 줄 요약

이 서비스는 게시판 UI, 백엔드 API, 원문 데이터베이스, AI API, MCP 원문 조회 도구가 연결된 근거 기반 AI 게시판입니다.

## 발표 대본

이 슬라이드는 전체 시스템 구조입니다.

사용자는 왼쪽의 React/Vite 프론트엔드에서 역사 질문을 작성하고, 게시글을 검색하고, 댓글이나 음성으로 토론합니다.

그 요청은 가운데의 FastAPI 백엔드로 전달됩니다. 백엔드는 단순히 CRUD만 처리하는 것이 아니라, 인증, 게시글 생성, 검색, AI Agent 실행, Realtime 음성 토론까지 담당합니다.

오른쪽에는 두 가지 외부/저장 계층이 있습니다. PostgreSQL과 pgvector는 사용자, 게시글, 댓글, 조선왕조실록 원문, 검색용 벡터 색인을 저장합니다. OpenAI API는 요약, 해석, 임베딩, Realtime 음성 응답에 사용됩니다.

아래의 MCP Article Tool은 이 프로젝트에서 중요한 역할을 합니다. 검색된 `article_id`를 기준으로 실록 원문을 다시 조회해서, AI가 답변을 만들 때 사용할 근거를 명확히 가져옵니다.

그래서 이 구조의 핵심은 "사용자 질문을 바로 LLM에 던지는 서비스"가 아니라, 게시판과 원문 DB, 검색, MCP 조회, AI 생성을 연결한 근거 기반 게시판이라는 점입니다.

## 짧게 말할 때

한 문장으로 말하면, 프론트엔드 게시판에서 들어온 질문을 FastAPI가 처리하고, PostgreSQL의 실록 원문과 pgvector 검색, OpenAI API, MCP 원문 조회 도구를 연결해서 근거 기반 AI 답변을 만드는 구조입니다.
