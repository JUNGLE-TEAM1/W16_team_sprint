# 03. 데이터 모델 / ERD 발표 대본

## 슬라이드 한 줄 요약

게시글, 원문 데이터, 검색 색인, 댓글, 음성 토론 로그를 역할별로 분리해 저장합니다.

## 발표 대본

이 슬라이드는 데이터 모델입니다.

왼쪽의 `users`는 사용자 계정 정보를 저장합니다. 사용자는 `posts`를 작성하고, `comments`를 남기고, 음성 토론 세션에도 참여합니다.

가운데의 `posts`가 서비스의 중심 테이블입니다. 여기에는 사용자의 질문뿐 아니라 AI 요약, 쉬운 해석, 추천 태그, 그리고 어떤 실록 기사를 근거로 사용했는지를 나타내는 `evidence_article_ids`가 함께 저장됩니다.

아래의 `comments`는 게시글에 달리는 토론 댓글입니다. 이 댓글들은 AI 토론 도우미가 후속 답변을 만들 때 맥락으로 사용할 수 있습니다.

오른쪽의 `annals_articles`와 `annals_chunks`는 실록 데이터를 나누어 저장합니다. `annals_articles`는 원문 기사 자체를 보존하는 테이블이고, `annals_chunks`는 검색을 위해 원문을 잘라 embedding vector와 함께 저장하는 색인 테이블입니다.

이렇게 원문 보존 테이블과 검색 색인 테이블을 나눈 이유는, 원문은 신뢰 가능한 형태로 유지하면서도 검색은 chunk 단위로 더 잘 되게 하기 위해서입니다.

맨 오른쪽의 `voice_sessions`와 `voice_messages`는 음성 토론을 위한 테이블입니다. 음성 토론이 시작되면 세션이 생기고, 사용자와 AI의 전사 내용, 라우팅 판단, 추가 근거 정보가 메시지로 저장됩니다.

정리하면, 이 데이터 모델은 게시글과 AI 결과, 원문 근거, 댓글, 음성 토론 기록을 분리해서 관리하면서도 게시글 상세 화면에서 다시 연결해 보여줄 수 있도록 설계되어 있습니다.

## 짧게 말할 때

핵심은 `posts`가 AI 결과와 근거를 연결하고, `annals_articles`는 원문 보존, `annals_chunks`는 검색 색인, `voice_messages`는 음성 토론 로그를 맡는다는 점입니다.
