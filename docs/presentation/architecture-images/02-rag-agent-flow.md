# 02. RAG / Agent 게시글 생성 흐름 발표 대본

## 슬라이드 한 줄 요약

사용자 질문이 검색, 원문 조회, 근거 선별, AI 생성을 거쳐 근거 기반 게시글 답변으로 저장되는 흐름입니다.

## 발표 대본

이 슬라이드는 게시글을 작성했을 때 내부에서 어떤 과정이 일어나는지를 보여줍니다.

먼저 사용자가 제목과 역사 질문을 작성합니다. 이 질문은 곧바로 LLM으로 가지 않고, 백엔드의 Agent 흐름으로 들어갑니다.

첫 단계는 Query 분석입니다. 여기서 질문 안에 있는 왕 이름, 시기, 사건, 주제 같은 단서를 추출하고, 검색에 더 적합한 형태로 query를 정리합니다.

다음은 Hybrid Search입니다. 이 단계에서는 벡터 검색과 키워드 검색을 함께 사용해서 조선왕조실록 기사 후보를 찾습니다.

그 다음 MCP 원문 조회 단계가 있습니다. 검색 결과로 나온 `article_id`를 사용해서 실제 실록 원문 payload를 다시 가져옵니다. 이 단계 덕분에 AI 답변이 추상적인 검색 결과가 아니라 실제 원문 기사에 기반하게 됩니다.

이후 LLM Rerank가 후보 근거 중에서 질문과 가장 관련 있는 기사를 선별합니다. 그리고 선택된 근거를 바탕으로 Draft 생성 단계에서 요약, 쉬운 해석, 추천 태그를 만듭니다.

마지막으로 Post 저장 단계에서 질문, AI 결과, 추천 태그, 근거 article_id, agent trace를 함께 저장합니다.

즉, 이 프로젝트의 핵심은 "질문 작성 → 검색 → 원문 확인 → 근거 선별 → AI 답변 생성 → 근거와 함께 저장"이라는 흐름입니다.

## 짧게 말할 때

이 흐름은 LLM에게 질문을 바로 보내는 구조가 아니라, 먼저 실록 원문을 찾고 MCP로 확인한 뒤, 선별된 근거만 사용해서 요약과 해석을 생성하는 구조입니다.
